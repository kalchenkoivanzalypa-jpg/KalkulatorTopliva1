# Данные для бота (координаты, станции, базисы)

## Что обычно есть в PDF/прайсах

В типовых **прайс-листах и PDF** чаще всего есть только **название базиса, город/регион и цена**.  
**Географические координаты** и **коды станций ЕСР** для **Тарифного руководства №4** там **почти никогда не лежат** как готовые поля.

Поэтому:

1. **Координаты базиса** (площадка / НПП) — из ваших таблиц (`Базисы с координатами.xlsx` и т.п.) или вручную.
2. **Станция отгрузки Ж/Д** у базиса — если известна: имя + при возможности **ЕСР** + координаты станции (можно совпадать с координатами базиса, если отгрузка с той же точки).
3. **Справочник станций** для населённых пунктов — отдельный файл/выгрузка (в репозитории есть **заготовка** `rail_stations_seed.csv` для тестов; для боя замените на полный справочник).

> Коды `esr_code` в `rail_stations_seed.csv` **условные** (для отладки UI). Для реального ТР №4 нужны **настоящие ЕСР** из вашего справочника РЖД.

## Импорт в БД

Из корня `fuel_bot`:

```bash
source venv/bin/activate
python3 -m db.migrate_sqlite_rail
python3 import_geo_data.py --seed-stations
```

При запуске `import_exchange_data.py` / импорта базисов из Excel колонки Ж/Д в таблице `basis` (`rail_station_name`, `rail_esr`, …) **добавляются сами**, если в старом `fuel_bot.db` их ещё не было.

Опционально — ваши Excel:

```bash
python3 import_geo_data.py --seed-stations \
  --stations-xlsx "/path/to/станции.xlsx" \
  --bases-xlsx "/path/to/Базисы с координатами.xlsx"
```

## Колонки Excel (гибкие имена)

**Базисы** — любые из названий (регистр не важен):

| Назначение        | Примеры имён колонок                                      |
|-------------------|-----------------------------------------------------------|
| Название          | `name`, `базис`, `название`, `basis`                      |
| Город             | `city`, `город`                                           |
| Широта/долгота    | `latitude`/`longitude`, `широта`/`долгота`, `lat`/`lon`   |
| Транспорт         | `transport`, `transport_type`, `тип`, `транспорт` (`rail` / `auto` / `жд` / `авто`) |
| Станция Ж/Д       | `rail_station`, `станция`, `station`                      |
| Код ЕСР           | `rail_esr`, `esr`, `еср`                                  |
| Коорд. станции    | `rail_latitude`, `rail_lat`, `широта_ст` и аналоги для lon |

**Станции** — аналогично: `name`/`станция`, `esr_code`/`еср`, `latitude`/`longitude`, опционально `settlement_name`/`город`.

## Тарифное руководство №4

Если у вас есть Python-модуль с расчётом километража между станциями по ЕСР:

- В `.env` укажите путь: `RZD_TR4_MODULE=/полный/путь/к/tr4.py`  
- В модуле должна быть функция **`distance_km(origin_esr, dest_esr)`** или **`tariff4_distance(...)`**.

Если модуль не задан или нет кодов — используется **оценка**: прямая между точками × коэффициент `RAIL_ROUTE_FACTOR` (по умолчанию 1.15).

## Бюллетень СПбМТСБ + ваши два Excel (актуальный пайплайн)

### Файлы

| Файл | Колонки | Назначение |
|------|---------|------------|
| **баба.xlsx** | `basis:`, `широта:`, `долгота:`, `transport`, `код еср:` | координаты, тип доставки, ЕСР → таблица `basis` |
| **Базисы поставок.xlsx** | `code`, `fuel_type`, `basis`, `transport`, `notes` | код инструмента, топливо, базис → `products` + `product_basis_prices` |
| **PDF бюллетень** | «Рыночная» (без запасных колонок) | обновление `current_price` по `instrument_code` |

Если в PDF **нет** «Рыночной» для кода — цена **не меняется** (позже можно подставлять прошлые бюллетени с датой).

### Первый раз / смена схемы БД

После добавления поля `instrument_code` пересоздайте таблицу цен (SQLite):

```bash
python3 -m db.rebuild_product_basis_prices
```
